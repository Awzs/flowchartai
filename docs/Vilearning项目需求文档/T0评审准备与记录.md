## 2. 评审要点
1. **PoC 可行性**：性能指标是否满足阶段 0 目标；fallback 方案是否可接受。
2. **Registry 接口**：是否满足流程图/思维导图共存需求；未来扩展点是否清晰。
3. **上下文契约**：字段是否满足 extend 模式与上下文工程需求；是否存在安全隐患。
4. **迁移计划**：建表结构、双写策略、回滚方案是否可执行；脚本 dry-run 结果。
5. **质量预案**：测试/监控范围是否覆盖成功标准；落地排期与责任人。
